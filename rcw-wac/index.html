Technical Notes

Inference model
Claude Haiku 4.5 (claude-haiku-4-5-20251001) — fast, cheap RAG synthesis
Embedding model
OpenAI text-embedding-3-small — 1536 dimensions, used at ingest and at query time
Chunk size
~800 tokens per section chunk (single RCW/WAC/USC/CFR section)
Retrieval
Top 8 results via Reciprocal Rank Fusion (k=60) of vector cosine + BM25 full-text lanes
Max output
2,000 tokens per response
Corpus sources
RCW (leg.wa.gov XML), WAC (leg.wa.gov XML), USC (Cornell LII), CFR (eCFR.gov XML)
Vector store
Supabase + pgvector, HNSW index (m=16, ef_construction=64)
Project repo
jweisenfeld/AI — rcw-wac/ (UI + proxy + ingestion)

HNSW Index Health Check

Run in Supabase → SQL Editor before and after a test session. A high cache-hit rate means the vector index lives in RAM — queries are fast. A low rate means it's hitting disk — consider upgrading to Micro compute (Project Settings → Infrastructure → Compute).

SELECT indexrelname, idx_blks_read AS disk_reads, idx_blks_hit AS cache_hits, round(idx_blks_hit::numeric / nullif(idx_blks_hit + idx_blks_read, 0) * 100, 1) AS cache_hit_pct FROM pg_statio_user_indexes WHERE relname = 'rcw_wac_chunks' ORDER BY indexrelname;
> 95%Index in RAM — optimal latency 80–95%Partially cached — acceptable after a cold start < 80%Hitting disk — upgrade to Micro compute tier
